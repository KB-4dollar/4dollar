# 일정

## ✅ Day1 -기획, 설계, 초기 셋팅

[기획]

- 벤치마킹 서비스 분석 -> 각자 서비스 하나씩 찾고 핵심기능 정리해서 공유하기
- 기능 정리 -> 기능명세서 작성

[설계]

- URL 설계(페이지구조) / 화면설계서 작성
- 컴포넌트 설계
- API 설계/json구조 정의

## ✅ Day2 -기본 기능 구현

[공통]

- 라우팅 구현
- 공통 레이아웃
- 기본 UI(반응형)
- API 모듈화 (axios)
- Pinia store 구성

[기본 기능]

- 내역 목록 조회 (GET)
- 수입/지출 등록 (POST)

## ✅ Day3 -CRUD완성 + 필터/정렬/페이지네이션

[기본 기능]

- 수입/지출 등록 (POST)
- 내역 수정 (PUT/PATCH)
- 내역 삭제 (DELETE)

[추가 기능]

- 목록 필터: 날짜, 카테고리, 수입/지출
- 목록 정렬, 페이지 네이션

## ✅ Day4 -대시보드,프로필 + 테스트

[기본 기능]

- 홈 대시보드
  - 재정 요약: 총 수입, 총 지출, 순이익
  - UI
  - 차트(선택)
- 프로필 관리
  - 사용자 정보: 이름, 이메일 등의 프로필 정보 표시 및 수정

[테스트]

- QA 및 수정
